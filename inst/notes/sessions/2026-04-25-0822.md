### Strategic decisions recorded in the chat (now binding)

1. **mfsusieR replaces fsusieR.** No `Imports: fsusieR`, no
   runtime calls into fsusieR. Routines we need (`remap_data`,
   `colScale`, `gen_wavelet_indx`, `init_prior.default`, smash /
   TI / HMM smoothers) ported into `R/utils_wavelet.R`,
   `R/prior_scale_mixture.R`, `R/post_processing.R`. fsusieR
   becomes a port source alongside `mvf.susie.alpha`.
   `wavethresh::wd` and `::wr` remain as math primitives.
2. **Three apple-to-apple equivalence contracts.** C1 (susieR
   scalar, `<= 1e-10`), C2 (fsusieR/susiF, `<= 1e-8`), C3
   (mvf.susie.alpha, `<= 1e-8`). When a port source has a known
   bug we fix, the corresponding test asserts the deviation
   explicitly; tolerance does not loosen.
3. **Decoupled post-processing API (Option B).** `mfsusie()` does
   NOT take a `post_processing` argument. `mf_post_smooth(fit,
   method = c("smash", "TI", "HMM"))` and `mf_credible_bands(fit,
   method = "TIWT", level = 0.95)` are separate functions. Fit
   stores per-modality residuals by default (`save_residuals =
   TRUE`); opt-out path takes `(X, Y)` explicitly.
4. **Two public entry points.** `mfsusieR::mfsusie(X, Y, pos,
   ...)` is the multi-modality entry; `mfsusieR::fsusie(Y, X,
   pos, ...)` is a thin wrapper for the single-phenotype case
   with `(Y, X)` argument order matching `fsusieR::susiF`.
5. **`fsusieR::EBmvFR` is out of scope.** Different model, no
   SuSiE structure. Marked at file level in
   `inst/notes/refactor-exceptions.md`.
6. **OpenSpec skill flow is binding for future design work.**
   `/openspec-explore` to clarify requirements, `/openspec-propose`
   to scaffold new changes, `/openspec-apply-change` to drive the
   review loop on tasks, `/openspec-archive-change` for the final
   merge. Hand-editing + `openspec validate` only for in-place
   refinement of an already-scaffolded change. Recorded in
   CLAUDE.md "Tools and when to use them" with a new "OpenSpec
   skill flow (binding for all future design work)" subsection.
