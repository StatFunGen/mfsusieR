# mf-covariate-adjustment capability

## ADDED Requirements

### Requirement: `mf_adjust_for_covariates()` is the public covariate-adjustment entry point

`mf_adjust_for_covariates()` SHALL be the public function for
adjusting a functional response `Y` for the effect of
covariates `Z`, with optional Frisch-Waugh-Lovell
residualization of a genotype matrix `X`. It SHALL support
two methods, `"wavelet_eb"` (default) and `"ols"`, and SHALL
NOT depend on any port-source package at runtime.

#### Scenario: signature

```
mf_adjust_for_covariates(Y, Z, X = NULL,
                         method = c("wavelet_eb", "ols"),
                         wavelet_filter_number = 1L,
                         wavelet_family        = "DaubExPhase",
                         max_iter              = 100L,
                         tol                   = 1e-3,
                         null_prior_weight     = 10,
                         init_pi0_w            = 1,
                         control_mixsqp        = list(verbose = FALSE,
                                                      eps = 1e-6,
                                                      numiter.em = 4L),
                         grid_mult             = sqrt(2))
```

The function SHALL return `list(Y_adjusted, X_adjusted,
fitted_func, sigma2, niter, converged, method)`.

`X_adjusted` SHALL be `NULL` when `X` is not supplied.

### Requirement: `wavelet_eb` matches the upstream EBmvFR-adjust formula bit-identically at defaults

`mf_adjust_for_covariates(Y, Z, method = "wavelet_eb")` SHALL
produce `Y_adjusted` and `fitted_func` fields that match the
equivalent fields produced by the upstream
`EBmvFR(Y, X = Z, adjust = TRUE)` routine at tolerance
`<= 1e-8`, when called with the default upstream-equivalent
configuration (`wavelet_filter_number = 1L`,
`wavelet_family = "DaubExPhase"`, `max_iter = 100L`,
`tol = 1e-3`, `null_prior_weight = 10`, `init_pi0_w = 1`,
default `control_mixsqp`, `grid_mult = sqrt(2)`).

#### Scenario: bit-identity sweep

For each combination of `n in {100, 200}`, `T in {64, 128}`,
`K in {2, 3, 5}` from a fixed seed, `tests/testthat/test_adjust_covariates.R`
SHALL assert
`expect_equal(ours$Y_adjusted, ref$Y_adjusted, tolerance = 1e-12)`
and `expect_equal(ours$fitted_func, ref$fitted_func, tolerance = 1e-12)`,
where `ref` is generated by the upstream port source. The test
SHALL be gated on `skip_if_not_installed("fsusieR")`.

#### Scenario: kept-upstream-bug for `sigma2`

`mf_adjust_for_covariates(method = "wavelet_eb")$sigma2` SHALL
match the upstream EBmvFR's `sigma2` field at tolerance
`<= 1e-8`, even though the upstream formula is missing the
`predictor_weights` factor and is therefore incorrect by the
manuscript's variance decomposition. The bug is preserved for
apple-to-apple reproducibility; `Y_adjusted` is not affected
because `sigma2` is dead-end in EBmvFR's outer loop. See
`inst/notes/refactor-exceptions.md` for the formal entry.

### Requirement: `ols` performs closed-form Frisch-Waugh-Lovell residualization

When `method = "ols"`, `mf_adjust_for_covariates()` SHALL
return:

- `Y_adjusted = (I - Z (Z'Z)^{-1} Z') Y`
- `X_adjusted = (I - Z (Z'Z)^{-1} Z') X` when `X` is supplied
- `fitted_func = (Z'Z)^{-1} Z' Y` (the OLS coefficient matrix)

The OLS path SHALL NOT call any wavelet-domain helpers or any
ash / mixsqp routines.

#### Scenario: orthogonality of OLS residuals

`max(abs(t(Z) %*% mf_adjust_for_covariates(Y, Z, method = "ols")$Y_adjusted))`
SHALL be `< 1e-12` for any well-conditioned `(Y, Z)` input.

#### Scenario: idempotency of OLS adjustment

For any well-conditioned `(Y, Z)` input,
`mf_adjust_for_covariates(mf_adjust_for_covariates(Y, Z, method = "ols")$Y_adjusted, Z, method = "ols")$Y_adjusted`
SHALL equal
`mf_adjust_for_covariates(Y, Z, method = "ols")$Y_adjusted` at
tolerance `<= 1e-12`.

### Requirement: unsupported preprocessing flags error rather than silently diverge

`mf_adjust_for_covariates()` SHALL emit a clear error when
called with `thresh_lowcount > 0` or `quantile_trans = TRUE`.
The error message SHALL state "not supported in v1" and
identify which feature.

#### Scenario: unsupported flag rejection

`mf_adjust_for_covariates(Y, Z, thresh_lowcount = 0.01)` SHALL
error with a message containing "not supported in v1".

`mf_adjust_for_covariates(Y, Z, quantile_trans = TRUE)` SHALL
error with a message containing "not supported in v1".

### Requirement: vignette uses `mf_adjust_for_covariates()` and does not name port-source packages

`vignettes/fsusie_covariates_adjustment.Rmd` SHALL use
`mf_adjust_for_covariates()` for all covariate-adjustment
calls. It SHALL NOT call any port-source function and SHALL
NOT mention `fsusieR`, `EBmvFR`, `mvf.susie.alpha`, or any
other port-source package by name in any prose, code chunk,
or output.

#### Scenario: vignette grep audit

A grep over `vignettes/fsusie_covariates_adjustment.Rmd` for
`fsusieR`, `EBmvFR`, `mvf.susie.alpha`, `plot_susiF`, or
`susiF` SHALL return zero matches.

### Requirement: vignette reproduces the upstream fine-mapping result on `Y_adjusted`

The vignette SHALL use the same simulation recipe and seed
as the upstream `Adjusting_covariate.Rmd` (genotype matrix
slice, covariate functions, target-signal SNP, noise
structure). After computing `Y_adjusted` via
`mf_adjust_for_covariates()`, the vignette SHALL run
`fsusie(Y_adjusted, X, pos)` and report PIPs, credible
sets, and effect curves. These outputs SHALL match the
equivalent outputs of the upstream vignette at numerical
tolerance `<= 1e-12` (apple-to-apple) on PIPs, credible
sets, alpha, and lbf, with `sigma2` allowed to deviate at
that same tolerance because of the `get_ER2.EBmvFR`
upstream bug we preserve under Pattern B.

#### Scenario: end-to-end reproduction

A test (`tests/testthat/test_adjust_covariates.R`) SHALL
run both the upstream and our pipelines on a fixed seed and
assert `expect_equal(ours_fit$alpha, ref_fit$alpha,
tolerance = 1e-12)` and analogous equality on
`mu, mu2, lbf, lbf_variable, KL, pip` and exact equality on
`fit$sets$cs`. The test SHALL be gated on
`skip_if_not_installed("fsusieR")`.
