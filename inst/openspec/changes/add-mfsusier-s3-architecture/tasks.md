# Implementation tasks

Each numbered group is one PR. Each PR has tests landing in the same
commit per CLAUDE.md. Estimated workload per PR: at most one focused
day; if a task feels larger, split it.

Phase 4 reference tests interleave with Phase 3 implementation: every
PR that adds a numerical routine also adds the matching paired-comparison
test against the relevant port source per the test-fidelity rule
in `inst/notes/test-fidelity-policy.md`.
Three apple-to-apple equivalence contracts hold the port together
(design.md D11): C1 vs `susieR::susie` at `<= 1e-10`, C2 vs
`fsusieR::susiF` at `<= 1e-8`, C3 vs `mvf.susie.alpha::multfsusie` at
`<= 1e-8`.

PR groups that copy code from a port source SHALL include a
port-quality audit substep per design.md D13 and the review-loop
methodology (`inst/notes/review-loop-methodology.md`, step 1
substep): run the `simplify` skill or an Explore subagent on the
ported routine before the reviewer pass, address findings, then
verify behaviour-preservation via the relevant equivalence test.
